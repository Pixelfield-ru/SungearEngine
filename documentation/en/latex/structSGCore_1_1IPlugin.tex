Virtual plugin destructor. \begin{DoxyNote}{Note}
Please note that if you create your own plugin, then you must implement this destructor in the structure of your plugin.~\newline
In it, you must remove all resources from all collections located directly in the engine, since this process does not happen automatically.~\newline
For example\+: you must unsubscribe from all events you subscribed to; you must delete all assets that were loaded by the plugin using the standard SGCore\+::\+Asset\+Manager instance (SGCore\+::\+Assets\+Manager\+::get\+Instance). 
\end{DoxyNote}
