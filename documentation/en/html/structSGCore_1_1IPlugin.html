◆ ~IPlugin()

virtual SGCore::IPlugin::~IPlugin ( )
virtualdefault

Virtual plugin destructor.

Note
Please note that if you create your own plugin, then you must implement this destructor in the structure of your plugin.
In it, you must remove all resources from all collections located directly in the engine, since this process does not happen automatically.
For example: you must unsubscribe from all events you subscribed to; you must delete all assets that were loaded by the plugin using the standard SGCore::AssetManager instance (SGCore::AssetsManager::getInstance).

◆ onConstruct()

virtual std::string SGCore::IPlugin::onConstruct ( const std::vector< std::string > &  args)
protectedpure virtual

A function that is guaranteed to be called immediately after calling the plugin structure constructor.
It is called in SGCore::PluginsManager::loadPlugin and also in SGCore::PluginsManager::reloadPlugin.
If you create your own plugin, then you must implement this function in the structure of your plugin.
When calling SGCore::PluginsManager::createPluginProject, the implementation of this function is generated automatically.

Parameters
argsArguments that will be passed to this function.
Using them you can make flags for the plugin.
Returns
String with an error when constructing the plugin.
In ANY implementation, the function SHOULD return an EMPTY string when loading the plugin WITHOUT ERROR.
